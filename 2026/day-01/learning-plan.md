# My 90-Day DevOps Learning Plan

## Current Level
Working professional with Linux and scripting background, looking to transition into a dedicated DevOps/SRE role.

## 3 Clear Goals
1. Deploy a production-grade application on Kubernetes with CI/CD, monitoring, and GitOps
2. Automate infrastructure provisioning using Terraform and Ansible
3. Build and contribute to an end-to- end observability stack (Prometheus, Grafana, Loki, OpenTelemetry)

## 3 Core DevOps Skills to Build
- Linux troubleshooting and automation (shell scripting, systemd, log analysis)
- CI/CD pipeline design and container orchestration (Docker, GitHub Actions, Kubernetes)
- Infrastructure as Code and configuration management (Terraform, Ansible)

## Weekly Time Budget
- Weekdays: 2–2.5 hours daily (after work)
- Weekends: 4–6 hours each day
- Total: ~22 hours/week

## Consistency Plan
- No zero days — even 30 minutes of reading counts
- Log daily progress in this repo and share on LinkedIn
- Revise previous week every Sunday

## Execution Blueprint
| Phase | Weeks | Focus |
|-------|-------|-------|
| Foundation | 1–3 | Linux, Networking, Shell Scripting, Git |
| Containers | 4–5 | Docker, Docker Compose, Docker Hub |
| CI/CD | 6–7 | GitHub Actions, DevSecOps |
| Orchestration | 8–9 | Kubernetes, Helm |
| IaC & Config | 10 | Terraform |
| Config Mgmt | 11 | Ansible |
| Observability | 12 | Prometheus, Grafana, Loki |
| Advanced & Final | 13 | EKS, GitOps, AI Agents, Portfolio |

Discipline > Motivation. Consistency > Perfection.
